JavaScript：声明提升（hoist）

2017-10-12   #声明提升 

读书笔记《你不知道的JavaScript（上卷）》提升。

自问自答

  • 问：声明提升是什么鬼？
  • 答：在JavaScript中，作用域中所有声明（包括变量声明和函数声明）会提升到改作用域的顶部，这就是声明提升。
  • 问：只有声明会提升？赋值操作会不会一起提升？
  • 答：只有声明会提升，赋值操作和其他逻辑还在原地，如果赋值操作也提升会改变代码的执行顺序，造成非常严重的破坏。
  • 问：该怎么理解这种声明提升呢？
  • 答：还记得编译原理的第一步么？碰到var a = 2;时，JavaScript会将其看成两个声明：
    1. var a; 编译阶段进行。
    2. a = 2; 执行阶段进行。

实例分析

a = 2;
var a;
console.log(a);         // 2

上面代码实际上是按照以下流程处理的：

var a;                  // 声明提前
a = 2;                  // 赋值操作留在原地
console.log(a);         // 2

自问自答

  • 问：函数表达式会提升？。
  • 答：函数声明会被提升，但函数表达式不会提升。

实例分析：函数声明提升

foo();
function foo() {
    console.log(a);
    var a = 2;
}

上面代码实际上是按照以下流程处理的：

function foo() {            // 函数foo提升
    var a;                  // 声明提升
    console.log(a);         // undefined
    a = 2;                  // 赋值操作留在原地
}
foo();

实例分析：函数表达式不提升

foo();
var foo = function bar() {};

上面代码实际上是按照以下流程处理的：

var foo;                    // 变量声明foo提升
console.log(typeof foo);    // undefined
foo();                      // TypeError: foo is not a function
foo = function bar() {};    // 函数表达式不提升
console.log(typeof foo);    //function

自问自答

  • 问：函数声明和变量声明都会被提升，他们的顺序是怎样的？
  • 答：函数声明会被提升到普通变量声明之前，如果函数声明和变量声明同名，变量声明会被忽略。

实例分析：函数先于变量被提升

foo();
var foo;
function foo() {
    console.log(1);
}
foo = function() {
    console.log(2);
};

上面代码实际上是按照以下流程处理的：

function foo() {
    console.log(1);
}
// var foo; 声明被忽略
foo();                  // 1
foo = function() {
    console.log(2);
};
foo();                  // 2

不要在同一个作用域中重复定义变量和函数，经常会导致各种奇怪的问题。

总结

  • JavaScript引擎会将var a = 2;看作两个声明，第一个是编译阶段var a , 第二个是执行阶段a = 2。
  • 所有的声明（变量和函数）都会被“移动”到各自作用域的最顶端，这个过程称为提升。
  • 声明本身会被提升，而包括函数表达式在内的赋值操作不会提升。
  • 要注意避免重复声明，尤其是当普通变量和函数声明混合在一起的时候，否则会引起很多危险的问题。

测试题

单选题：下面代码的运行结果是什么？

var name = "world";
(function() {
    if (typeof name === 'undefined') {
        var name = 'Jack';
        console.log('GoogleBy: ' + name);
    } else {
        console.log('Hello: ' + name);
    }
})();
  1. GoogleBy Jack
  2. Hello Jack
  3. Hello undefined
  4. Hello world

分析：题目考查变量提升，转换为更易理解的代码：

var name = "world";
(function() {
    var name;
    if (typeof name === 'undefined') {
        name = 'Jack';
        console.log('GoogleBy: ' + name);
    } else {
        console.log('Hello: ' + name);
    }
})();

答案：A
