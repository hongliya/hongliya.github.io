JavaScript：null和undefined的区别

2018-01-16   #数据类型 

相同点

1.Null和Undefined都只有一个值

  • Null的值是null
  • Undefined的值是undefined

2.参与判断的时候都返回false

!null           // true
!undefined      // true

3.Null和Undefined都没有自己的方法

不同点

1.null是关键字，undefined不是关键字

var null;       // Uncaught SyntaxError: Unexpected token null
var undefined;  // undefined

2.声明一个变量但未初始化时，该变量的默认值是undefined，null是已经初始化的一个空对象的指针

var a;          // undefined
var b = null;   // null

3.typeof检测类型时，null会返回”object”，undefined会返回”undefined”

typeof null;        // "object"
typeof undefined;   // "undefined"

4.null转换成数字是0，undefined转换成数字是NaN

Number(null);       // 0
Number(undefined);  // undefined

开发中的注意事项

1.声明变量时尽量初始化一个值（但不要初始化为undefined），因为使用typeof操作符检测声明未赋值的变量和未声明的变量返回的结果都是”undefined”。

var a;
typeof a; // "undefined"
typeof b; // "undefined" 注意b并未声明

3.

2.判断一个值是否存在时，可以使用 value == null，因为开发中一般后台返回数据如果没有值会返回null，但如果明确知道要判断的值是null或undefined还是用 ===。

var a;
a == null;          // true
