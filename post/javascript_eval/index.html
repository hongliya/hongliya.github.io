JavaScript：欺骗词法作用域之eval

2017-11-06   #eval 

读书笔记《你不知道的JavaScript（上卷）》词法作用域。

欺骗词法作用域会导致性能下降，不推荐使用。

但仍然要了解下原理。

自问自答

  • 问：eval是怎么欺骗（修改）词法作用域的呢？
  • 答：eval可以接受一个字符串作为参数，并将这个参数偷偷的放进词法作用域中，就好像写代码的时候它就在那里一样。举个例子：把eval比做一个超能力者，它可以修改一个人（名字叫A）的记忆，把一段别人的记忆插入到A的大脑中，让A以为这是自己的记忆。

实例分析

// 正常代码
function foo(str, a) {
    console.log(a, b); // 1, 2
}
var b = 2;
foo('var b = 3;', 1);

正常代码分析：

全局作用域 销毁
foo作用域 销毁
console作用域 销毁 一系列操作 输出结果 1,2 console作用域 创建
调用console.log(a, b) a = 1 str = 'var b = 3' foo作用域 创建
调用foo('var b = 3;', 1) b = 2 全局作用域 创建

参数’var b = 3;‘并未改变任何作用域，只是作为一个普通的字符串参数传进去而已，打印下str仍然是字符串’var b = 3;‘。

// 使用eval欺骗词法作用域
function foo(str, a) {
    eval(str); // 欺骗
    console.log(a, b); // 1, 3
}
var b = 2;
foo('var b = 3;', 1);

使用eval欺骗词法作用域代码分析：

全局作用域 销毁
foo作用域 销毁
console作用域 销毁 一系列操作 输出结果 1,3 console作用域 创建
调用console.log(a, b) a = 1 b = 3（注意这里b变成了3） 调用eval(str) foo作用域 创建
调用foo('var b = 3;', 1) b = 2 全局作用域 创建

这里调用eval后也是要进入到eval的作用域中，这里就不画了，参数’var b = 3;‘这段代码在foo的作用域中声明了一个新的变量b，并遮蔽了全局作用域中的同名变量，对已经存在的foo词法作用域进行了修改。

在严格模式的程序中，eval()在运行时有自己的词法作用域，其中的声明无法修改所在作用域。

// 严格模式中
function foo(str) {
    'use strict';
    eval(str);
    console.log(a); // ReferenceError: a is not defined
}
foo('var a = 2');
全局作用域 销毁
foo作用域 销毁
console作用域 销毁 输出a，没找到，报错Reference: a is not defined console作用域 创建
调用console.log(a)
eval作用域 销毁 a = 2 eval作用域 创建
调用eval(str) foo作用域 创建
调用foo('var a = 2') 全局作用域 创建

在程序中动态生成代码的使用场景非常罕见，因为它所带来的好处无法抵消性能上的损失。
